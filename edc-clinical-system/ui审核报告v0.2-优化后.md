## 详细发现（按严重程度）

### Medium 中优先级问题（剩余）

#### 1. 未规范化页面仍有硬编码颜色
**位置**：`VisitConfig.tsx`（12 处）、`CRFDesigner.tsx`（9 处）、`CRFForm.tsx`（8 处）、`SubjectList.tsx`（3 处）

**示例**：
```tsx
// VisitConfig.tsx:99
{isLinked && <CheckSquareOutlined style={{ color: '#1890ff' }} />}

// CRFDesigner.tsx:110
<div style={{ fontSize: 20, marginBottom: 4, color: '#1890ff' }}>
```

**影响**：这些页面不支持主题切换，深色模式下可能显示异常
**建议**：使用 `/normalize` 继续规范化剩余页面

#### 2. 部分表格缺少键盘导航
**位置**：`SubjectList.tsx`（操作按钮无 aria-label，表格行无键盘支持）

**当前状态**：
```tsx
// 缺少 tabIndex、role、onKeyDown
<Button type="text" icon={<EyeOutlined />} />
```

**建议**：参考已规范的 QueryManagement 和 ProjectList 实现

#### 3. 空状态未处理
**位置**：多个表格组件

**影响**：数据为空时用户体验差
**建议**：添加空状态提示组件

---

### Low 低优先级问题

#### 4. 组件缺少 memo 优化
**位置**：`SubjectList.tsx`、`VisitConfig.tsx`、`SystemConfig.tsx`

**影响**：大数据量时可能产生不必要的重渲染
**建议**：使用 `React.memo` 和 `useCallback` 优化

#### 5. 抽屉关闭无确认
**位置**：`CRFForm.tsx`、`QueryManagement.tsx`

**影响**：用户可能丢失未保存的数据
**建议**：添加关闭确认对话框

#### 6. 表单错误提示待完善
**位置**：`DataExport.tsx`、`SystemConfig.tsx`

**影响**：用户可能不清楚错误原因
**建议**：完善表单验证和错误提示
