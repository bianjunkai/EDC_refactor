## 一、业务背景与目标

### 1.1 行业背景

临床研究电子数据采集系统（EDC，Electronic Data Capture）用于临床试验中病例报告表（CRF）的电子化设计、数据录入、审核、质疑处理及稽查留痕，符合 GCP（药物临床试验质量管理规范）对数据真实性、完整性和可追溯性的要求。

### 1.2 业务目标

- **数据采集**：支持多中心、多受试者、多访视的 CRF 数据录入与导入  
- **流程管理**：项目立项、审批、进行、锁库的全生命周期管理  
- **质量保障**：质疑-回复闭环、稽查留痕、PI 签字、数据锁库  
- **合规追溯**：登录日志、项目留痕、中心留痕、数据导出（Excel/PDF/SAS/稽查痕迹）

### 1.3 目标用户

| 用户类型 | 描述 | 典型场景 |
|----------|------|----------|
| 研究机构管理员 | 平台配置、用户与角色、中心管理 | 系统初始化、权限分配 |
| 项目管理员 / PI | 项目管理、受试者、CRF 配置、数据审核 | 立项、配置访视与表单、PI 签字 |
| 数据录入员 | CRF 数据录入、质疑发起与回复 | 日常数据采集 |
| 审核人员 | 项目审核、质疑处理 | 审批流程、质疑闭环 |
| 稽查人员 | 稽查留痕、数据导出 | 合规稽查、数据导出 |
| 监查员（SDV） | SDV 留痕 | 源数据核查 |

## 三、核心业务流程

### 3.1 项目生命周期

```
立项中(1) → 审批中(2) → [已驳回(3)] 或 [进行中(4)]
                              ↓
                         PI 签字
                              ↓
                         已锁库(7) [或 解除锁库 → 进行中(4)]
```

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| 1 | 立项中 | 新建后待提交 |
| 2 | 审批中 | 已提交，等待审批 |
| 3 | 已驳回 | 审批驳回，可修改后重新提交 |
| 4 | 进行中 | 审批通过，可数据录入、质疑、稽查 |
| 5 | 已完成 | 项目完结（业务侧可扩展） |
| 6 | 已作废 | 项目废弃 |
| 7 | 已锁库 | 数据锁定，不可修改 |

### 3.2 项目审核流程

- 每个项目绑定一个**审核流程**（来自「审核流程」模块）  
- 审核流程由多步骤组成，每步对应一个审批角色  
- 项目创建时选择审核流程，生成 `proCheckList`（项目审核节点列表）  
- 审核人员按步骤依次审批，可「通过」或「驳回」  
- 审核通过后项目进入「进行中」；驳回后进入「已驳回」

### 3.3 CRF 数据录入流程

1. 项目配置访视计划（Play），每个访视绑定若干 CRF  
2. 受试者入组后，按访视计划分配 CRF 填写任务  
3. 数据录入员进入对应 CRF 表单进行录入  
4. 支持手工录入、Excel 导入；可暂存，最终提交  
5. 数据修改需填写修改原因（`sremark`），留痕可查  
6. 锁库后数据不可再修改

### 3.4 质疑流程

| 环节 | 说明 |
|------|------|
| 发起质疑 | 对某条 CRF 数据发起质疑，填写质疑内容 |
| 回复质疑 | 被质疑方可回复 |
| 完成质疑 | 质疑方确认后标记为「完成」 |
| 撤销质疑 | 质疑方可撤销未完成的质疑 |

质疑状态：1=待处理，2=已完成，3=已撤销

### 3.5 PI 签字与锁库

- **PI 签字**：项目进行中，具备 `pro_pi` 权限的用户可执行 PI 签字，表示数据经 PI 审核确认  
- **锁库**：PI 签字后，具备 `pro_db` 权限的用户可执行「锁库」，锁定项目所有数据，禁止修改  
- **解除锁库**：必要时可解除锁库，恢复数据修改（需严格管控）

### 3.6 稽查留痕类型

| 类型 | 说明 |
|------|------|
| 受试者留痕 | 受试者入组、出组等操作记录 |
| 表单留痕 | CRF 数据增删改记录 |
| 项目状态留痕 | 立项、审批、锁库等状态变更 |
| SDV 留痕 | 监查员源数据核查记录 |
| PI 审查留痕 | PI 签字相关记录 |
| 数据导出留痕 | Excel/PDF/SAS 等导出记录 |

**控件通用属性**：`lab`(变量名)、`sasKey`/`sasLab`(SAS 导出用)、`showTitle`(1=不显示标题 2=显示)、`showHide`(1=显示 0=隐藏)、`ismust`(必填)

### 4.0.2 页面控制（pageConfig）

控件可配置**显示/隐藏条件**，分为三类：

| ctype | 说明 |
|-------|------|
| `in` | 内部控制：当前表单内某控件的值满足条件时，控制目标显示/隐藏 |
| `out` | 外部控制：其他 CRF 的某控件值满足条件时，控制目标显示/隐藏 |
| `base` | 基线控制：基线 CRF 的某控件值满足条件时，控制目标显示/隐藏 |

**条件类型**：`where` 支持 `equal`(等于)、`in`(存在于)、`have`(包含)；`show` 为 1=显示、0=隐藏。

### 4.0.3 数据字典关联

下拉框、自定义表格列等可关联「数据字典」：

- 选择**关联表**（dTableName）、**关联字段**（dTableField）
- 支持**筛选条件**：fname、fif(eq/have)、fvalue；支持 `and`/`or` 组合
- 特殊变量：`@EMPI_ID` 表示受试者主索引号，用于按受试者过滤字典数据

### 4.0.4 访视配置明细

| 配置项 | 说明 |
|--------|------|
| 阶段-筛选期 | `screeningFlag`：0=否，1=是；筛选期阶段标记 |
| 访视次数 | `playLoop`：1=可循环访视，2=单次访视 |
| 循环次数 | `loopTimes`：可循环时生效；0 表示不限制次数 |
| 时间窗 | `playWindow`：访视时间窗（天数） |
| TBO | `playTBO`：访视时间窗相关参数 |
| 开始/结束 | `playSatrt`/`playEnd`：访视序号或时间相关 |
| CRF 绑定 | 一个访视可绑定多个 CRF，支持排序、筛选 |

### 4.0.5 受试者状态（patState）

| 值 | 含义 | 统计归类 |
|----|------|----------|
| 2 | 已入组 | patIng(进行中) / patNo(未分配访视) |
| 3 | 完成 | patEnd |
| 4 | 脱落 | patTL |
| 5 | 退出 | patExit |
| 6 | 剔除 | patTC |
| 7 | 失访 | patSF |

仅 `patState=2` 时可进行 CRF 数据录入；其他状态为出组状态。

### 4.0.6 随机化业务

- **前置条件**：项目 `randomFlag=1`（启用随机）
- **生成随机表**：按试验分组总人数、随机码前缀、试验组/对照组名称生成
- **扫码随机**：通过随机号或筛选号扫码完成随机入组
- **约束**：每个项目仅可生成一次随机表（`showCreateBtn` 控制按钮可用性）

### 4.0.7 数据导出类型

| 类型 | 说明 |
|------|------|
| Excel 导出 | 结构化 CRF 数据导出 |
| PDF 导出 | 按受试者/筛选号导出 PDF 报告 |
| 稽查痕迹导出 | 稽查留痕数据导出 |
| 数据结构导出 | 表结构、字段说明等 |
| SAS 导出 | 符合 SAS 格式的数据导出 |
| ODM 导出 | 当前禁用（label=5） |

### 4.0.8 项目标签（proTag）

项目支持**自定义扩展字段**：proTagType、proTagName、proTagValue、proTagShow；可增删改，用于扩展项目基础信息。

### 4.0.9 用户与密码规则

- 密码规则：≥8 位，同时包含大小写字母、数字、特殊字符
- 用户可绑定科室（deptId）
- 用户可上传签名图片，用于 PI 签字等场景

### 4.0.10 暂存与提交

- CRF 支持**暂存**（selectCrfDataTag），未提交前可多次暂存
- 提交后数据落库，修改需填写修改原因（sremark）

### 4.0.11 CRF 表单类型与状态

| 属性 | 值 | 含义 |
|------|-----|------|
| crfType | 1 | 私有 |
| crfType | 2 | 公开 |
| crfState | 1 | 编辑中（可设计、可发布） |
| crfState | 2 | 已发布（可关联项目、不可直接编辑结构） |
| crfState | 3 | 已禁用 |

项目 CRF 配置中仅可选择 crfState=2（已发布）的表单。

### 4.0.12 受试者与访视进度

- 受试者当前访视：`playId` 为空表示未分配访视；有值表示当前所处访视
- 进度计算：`ingP` = 当前访视在计划中的序号 / 计划总数 × 100
- 筛选号：`patEmpi`，用于受试者唯一标识与导出
- 仅 `patState=2` 且 `web_pi!=1`（未锁库）时可录入 CRF 数据

### 4.0.13 首页交互

- 卡片点击：立项中(1)、审核中(2)、已驳回(3)、进行中(4)、已锁库(7) → 跳转项目管理并带状态筛选
- 未解决质疑(11) → 跳转质疑管理
- 项目卡片(12) → 跳转项目管理并预选该项目
- 状态通过 sessionStorage 传递：`open_pro_state`、`open_question_state`、`proData`

### 4.0.14 签到（/sing）

- 独立于登录的签到入口
- 用于现场签到等场景

### 4.0.15 表单防护（formGuard）

- 离开 CRF 编辑/录入页面前，若有未保存数据，弹窗二次确认
- 路由守卫中统一拦截，`formGuardStore.hasUnsavedForm()` 为 true 时阻止跳转

### 4.0.16 质疑可见性与操作权限

- 项目 `proState=4`（进行中）时可发起/回复质疑
- `question_in`：发起质疑；`question_back`：回复质疑
- 质疑发起人可完成/撤销；被质疑方（数据录入人）可回复

## 六、业务规则与约束

### 6.1 项目相关

- 立项中、审批中、已驳回：可编辑基础信息、配置；不可数据录入  
- 进行中：可数据录入、质疑、稽查、导出；PI 签字后可锁库  
- 已锁库：数据只读，不可修改、不可解除 PI 签字  
- 项目必须绑定审核流程才能提交审批  

### 6.2 数据相关

- 数据修改需填写修改原因，用于稽查追溯  
- 数据来源：1=手工录入，2=导入  
- 锁库后禁止新增、修改、删除 CRF 数据  

### 6.3 质疑相关

- 只有质疑发起人可回复/完成/撤销  
- 被质疑方（数据录入人）可回复  
- 完成前可撤销  

### 6.4 权限相关

- 菜单权限控制是否显示菜单入口  
- 操作权限控制按钮级操作（如锁库、PI 签字）  
- 无权限时对应功能不可见或禁用

## 七、数据实体（业务视角）

### 7.1 项目域

- **项目**：基础信息、状态、审核流程、中心、标签  
- **试验分组**：组名、受试者数量  
- **访视阶段**：阶段名、顺序、筛选标志  
- **访视计划**：访视名、关联 CRF、循环配置、时间窗  
- **受试者**：入组信息、分组、访视完成情况  
- **项目审核节点**：审核步骤、审核人、审核状态、审核意见  

### 7.2 CRF 域

- **表单类别**：分类  
- **CRF 表单**：表名、名称、类型、状态  
- **表单控件**：控件类型、校验、显示隐藏、数据字典关联  
- **CRF 数据**：录入值、录入人、录入时间、数据来源、质疑标识  

### 7.3 质疑域

- **质疑**：质疑内容、发起人、回复内容、状态、关联 CRF 数据  

### 7.4 系统域

- **用户**：账号、姓名、科室、角色、签名  
- **角色**：角色名、权限配置  
- **中心**：中心名、编码  
- **数据字典**：表、字段  
- **审核流程**：流程名、审核步骤（角色）

## 十、典型业务场景（用例简述）

| 场景 | 前置条件 | 主要步骤 |
|------|----------|----------|
| 新建项目并审批 | 有审核流程、中心 | 填写基础信息→选择审核流程、中心→保存→完成并发起审批→审核人通过 |
| 第三方项目创建 | 启用第三方接入、有外部项目源 | 通过「发起新项目」选择外部项目→使用 ProBaseOut 填写/同步基础信息 |
| 受试者入组与录入 | 项目进行中、已配置访视与CRF | 入组受试者→选择分组→分配访视→进入CRF录入数据→暂存/提交 |
| 质疑处理 | 项目进行中、有质疑权限 | 对数据发起质疑→录入人回复→质疑方确认完成 |
| 数据锁定 | 项目进行中、已完成PI签字 | PI签字→锁库→数据只读 |
| 随机入组 | 项目启用随机、已生成随机表 | 扫码或输入随机号/筛选号→完成随机入组 |
| 数据导出 | 项目进行中/已锁库、有导出权限 | 选择导出类型→选择项目/受试者→导出Excel/PDF/SAS等 |
| CRF设计 | 有CRF管理权限 | 新建表单→拖拽控件→配置属性、页面控制、字典→发布 |
