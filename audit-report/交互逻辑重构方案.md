# EDC临床研究数据采集系统 - 交互逻辑重构方案

**文档版本**：v1.0
**日期**：2026-03-16
**问题提出**：Header导航与项目详情页功能边界模糊，用户交互路径混乱

---

## 一、问题诊断

### 1.1 当前架构的问题

#### 问题1：功能入口重复且含义不同

| 功能名称 | Header导航（全局视角） | 项目详情页（项目视角） | 用户困惑 |
|---------|---------------------|-------------------|---------|
| **受试者管理** | 查看所有项目的受试者 | 管理当前项目的受试者 | 不知道应该在哪里入组 |
| **CRF管理** | 设计CRF表单（全局模板） | 启用/配置项目CRF | 不知道在哪里创建表单 |
| **访视配置** | 独立的访视配置入口 | 项目内的访视期配置 | 不知道在哪里配置访视 |
| **质疑管理** | 查看所有质疑 | 查看项目内质疑 | 功能重复，数据范围不同 |
| **数据导出** | 全局数据导出 | 项目内数据导出 | 导出范围不明确 |

#### 问题2：配置层级混乱

```
当前混乱的层级关系：

Header导航（误以为全局配置）
├── 受试者管理  →  实际只是跨项目查询
├── CRF管理     →  实际是表单设计器（全局模板）
├── 访视配置    →  实际应该是项目级配置
└── 质疑管理    →  实际是跨项目查询

ProjectDetail（项目内配置）
├── CRF配置     →  选择项目使用哪些CRF
├── 数据采集    →  查看项目采集进度
├── 数据导出    →  导出当前项目数据
└── 质疑管理    →  查看当前项目质疑
```

#### 问题3：权限控制不清晰

当前权限设计没有区分：
- **系统级权限**：配置全局模板、字典、用户等
- **项目级权限**：在项目内进行操作
- **数据级权限**：查看跨项目数据

### 1.2 用户使用场景分析

#### 场景1：数据录入员（研究护士）
> "我今天要为项目A的受试者张三填写第3次访视的CRF"

**期望路径**：项目 → 受试者 → 访视 → CRF填写
**当前困惑**：Header有"受试者管理"，但进去后不知道选哪个项目

#### 场景2：项目经理（PI）
> "我要为项目B配置使用的CRF表单和访视计划"

**期望路径**：项目 → 项目配置 → CRF/访视设置
**当前困惑**：Header有"CRF管理"和"访视配置"，但配置后需要再到项目里关联

#### 场景3：数据管理员
> "我要查看所有项目中待处理的质疑"

**期望路径**：工作区 → 待办质疑
**当前困惑**：Header有"质疑管理"，但不知道是否包含所有项目

#### 场景4：系统管理员
> "我要配置全局的CRF模板和标准字典"

**期望路径**：系统配置 → 模板管理/字典管理
**当前困惑**：Header有"CRF管理"（以为是模板），但进去是设计器

## 二、重构方案

### 2.1 新的信息架构

将功能明确划分为三个层级：

```
┌─────────────────────────────────────────────────────────────┐
│                     新的信息架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【第一层】工作区（数据操作层）                              │
│  ├── 首页（Dashboard）- 我的项目、待办事项                 │
│  ├── 我的工作 - 待录入CRF、待处理质疑、待审核项目          │
│  └── 数据查询 - 受试者查询（跨项目）、质疑查询（跨项目）   │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【第二层】项目管理（核心业务层）                            │
│  ├── 项目列表                                              │
│  └── 项目详情（包含项目内所有功能）                        │
│       ├── 概览                                             │
│       ├── 受试者管理（入组、随机化、填写CRF）              │
│       ├── CRF配置（选择项目使用的CRF模板）                 │
│       ├── 访视配置（配置项目的访视计划）                   │
│       ├── 数据管理（导出、质疑、留痕）                     │
│       └── 项目设置（成员、权限、审核流程）                 │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【第三层】系统配置（全局管理层）                            │
│  ├── 全局CRF库（CRF模板设计、管理）                        │
│  ├── 标准字典库（MedDRA、WHODrug等）                       │
│  ├── 组织机构（用户、角色、中心管理）                      │
│  └── 系统设置（日志、备份、参数）                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 新的Header导航设计

```
Header导航（简化，聚焦高频操作）
├─ 🏠 首页
├─ 📋 项目（项目列表入口）
├─ 📝 我的工作（个人工作区）
│   ├─ 待录入数据
│   ├─ 待处理质疑
│   └─ 待审核项目
├─ 🔍 数据查询（跨项目查询）
│   ├─ 受试者查询
│   ├─ 质疑查询
│   └─ 数据导出
└─ ⚙️ 系统配置（管理员可见）
    ├─ CRF模板库
    ├─ 标准字典库
    ├─ 用户管理
    └─ 系统日志
```

### 2.3 项目详情页重构

ProjectDetail标签页简化为6个核心标签：

```
项目详情页标签结构
├─ 📊 概览 - 项目进度、关键指标、入组趋势
├─ 👥 受试者 - 受试者列表、入组、随机化、填写CRF
├─ 📝 CRF配置 - 选择项目使用的CRF、配置访视-CRF矩阵
├─ 📅 访视计划 - 访视期配置、窗口期设置
├─ 📊 数据管理 - 质疑管理、数据导出、稽查留痕
└─ ⚙️ 项目设置 - 基本信息、成员、权限、审核流程
```

**关键变化**：
1. **删除"CRF表单设计"功能** - 移至系统配置的"CRF模板库"
2. **删除"模板市场"** - 移至系统配置的"CRF模板库"
3. **合并"数据采集"到"数据管理"**
4. **合并"统计报表"到"概览"**

### 2.4 功能迁移对照表

| 原位置 | 原功能 | 新位置 | 新功能 | 说明 |
|--------|--------|--------|--------|------|
| Header-受试者管理 | 跨项目受试者列表 | Header-数据查询-受试者查询 | 全局受试者搜索 | 改为查询入口 |
| Header-CRF管理 | CRF设计器+模板市场 | Header-系统配置-CRF模板库 | CRF模板管理 | 仅限管理员 |
| Header-访视配置 | 访视配置 | 移除 | - | 移至项目详情页 |
| Header-质疑管理 | 跨项目质疑 | Header-我的工作-待处理质疑 | 个人待办质疑 | 聚焦待办 |
| Header-数据导出 | 全局导出 | Header-数据查询-数据导出 | 跨项目导出 | 需要高级权限 |
| Header-字典管理 | 字典管理 | Header-系统配置-标准字典库 | 字典管理 | 保持不变 |
| ProjectDetail-CRF配置 | 简单列表 | ProjectDetail-CRF配置 | 从模板库选择+配置 | 增强功能 |
| ProjectDetail-访视期 | 无独立标签 | ProjectDetail-访视计划 | 独立标签页 | 新增 |

### 2.5 用户角色与功能映射

#### 数据录入员（研究护士）
```
可见导航：
├── 🏠 首页
├── 📋 项目（进入项目后操作）
└── 📝 我的工作
    ├── 待录入数据（按项目分组）
    └── 待处理质疑（与我相关的）

主要操作路径：
首页 → 项目 → 选择项目 → 受试者 → 填写CRF
```

#### 项目经理（PI）
```
可见导航：
├── 🏠 首页
├── 📋 项目
├── 📝 我的工作
├── 🔍 数据查询（受限）
└── ⚙️ 系统配置（受限）
    └── CRF模板库（只读）

主要操作路径：
首页 → 项目 → 选择项目 → CRF配置/访视计划/项目设置
```

#### 系统管理员
```
可见导航：全部

主要操作路径：
系统配置 → CRF模板库/标准字典库/用户管理
```

## 三、权限控制新策略

### 3.1 三级权限模型

```
权限层级
├─ 系统级权限（System Level）
│   ├─ 全局CRF模板管理
│   ├─ 标准字典库管理
│   ├─ 用户/角色管理
│   └─ 系统配置
│
├─ 项目级权限（Project Level）
│   ├─ 项目查看
│   ├─ 受试者管理（入组、编辑）
│   ├─ CRF配置（选择模板）
│   ├─ 访视配置
│   ├─ 数据导出
│   └─ 项目设置（成员、权限）
│
└─ 数据级权限（Data Level）
    ├─ 跨项目数据查询
    ├─ 全局质疑查看
    └─ 全局报表查看
```

### 3.2 权限矩阵（简化版）

| 功能 | 系统管理员 | 项目经理 | 数据录入员 | 审核员 | 稽查员 |
|------|----------|---------|----------|-------|-------|
| **系统级** |||||
| CRF模板库管理 | ✅ | 只读 | ❌ | ❌ | ❌ |
| 标准字典库管理 | ✅ | 只读 | ❌ | ❌ | ❌ |
| 用户/角色管理 | ✅ | ❌ | ❌ | ❌ | ❌ |
| **项目级** |||||
| 受试者入组 | ✅ | ✅ | ✅ | ❌ | ❌ |
| CRF配置（选模板） | ✅ | ✅ | ❌ | ❌ | ❌ |
| 访视配置 | ✅ | ✅ | ❌ | ❌ | ❌ |
| 数据导出 | ✅ | ✅ | 受限 | ✅ | ✅ |
| **数据级** |||||
| 跨项目查询 | ✅ | 受限 | 仅自己的 | 仅审核相关 | ✅ |
| 全局报表 | ✅ | 仅自己的项目 | ❌ | ❌ | ✅ |

## 四、关键页面交互设计

### 4.1 我的工作页面

```
我的工作（个人工作区）
├─ 待录入数据（按项目分组）
│   └─ 项目A
│       ├─ 受试者001 - 访视3 - CRF待填写
│       └─ 受试者002 - 访视2 - CRF待填写
├─ 待处理质疑
│   ├─ 质疑#123 - 项目A - 需回复
│   └─ 质疑#124 - 项目B - 需回复
└─ 待审核项目（仅审核员）
    ├─ 项目C - 待审核
    └─ 项目D - 待审核
```

### 4.2 数据查询页面

```
数据查询（跨项目查询，需要高级权限）
├─ 受试者查询
│   ├─ 筛选：项目、中心、状态、日期

├─ 质疑查询
│   ├─ 筛选：项目、状态、发起人
│   └─ 结果列表（跨项目）
└─ 数据导出
    ├─ 选择项目、CRF、访视
    └─ 导出配置
```

### 4.3 项目详情-CRF配置页面

```
CRF配置（从模板库选择）
├─ 已选CRF列表（当前项目使用的）

│   └─ 打开模板选择器（从全局CRF模板库选择）
└─ 访视-CRF矩阵配置
    └─ 配置每个访视期需要填写哪些CRF
```

## 五、实施建议

### 5.1 实施优先级

#### P0 - 立即实施（核心架构调整）
1. **Header导航重构**
   - 移除"受试者管理"、"访视配置"独立入口
   - 新增"我的工作"、"数据查询"入口
   - 将"CRF管理"移至"系统配置"下

2. **ProjectDetail标签页调整**
   - 合并标签页为6个核心标签
   - 将访视配置提升为独立标签
   - 在CRF配置中添加"从模板库选择"功能

#### P1 - 重要实施（功能增强）
1. **新增"我的工作"页面**
   - 聚合当前用户的待办事项
   - 按项目分组展示

2. **新增"数据查询"页面**
   - 跨项目查询功能
   - 高级筛选和导出

3. **权限模型调整**
   - 区分系统级/项目级/数据级权限
   - 调整权限检查逻辑

#### P2 - 优化实施（体验提升）
1. **CRF模板库页面**
   - 合并CRF设计器和模板市场
   - 提供模板分类和搜索

2. **面包屑和导航优化**
   - 明确显示当前位置层级
   - 提供快捷返回路径

### 5.2 数据迁移

当前系统的数据不需要迁移，但需要：
1. 更新路由配置
2. 更新权限配置
3. 更新面包屑配置
4. 更新菜单权限映射

### 5.3 用户培训要点

1. **CRF设计的位置变化**
   - 原来：Header → CRF管理 → 表单设计
   - 现在：系统配置 → CRF模板库 → 设计模板

2. **项目配置流程**
   - 第一步：系统配置 → CRF模板库 → 创建模板
   - 第二步：项目 → CRF配置 → 选择模板
   - 第三步：项目 → 访视计划 → 配置访视

3. **日常工作入口**
   - 数据录入：我的工作 → 待录入数据
   - 质疑处理：我的工作 → 待处理质疑
   - 跨项目查询：数据查询

## 六、总结

### 6.1 核心变化

| 维度 | 当前设计 | 新设计 |
|------|---------|--------|
| **导航层级** | 扁平化，所有功能在Header | 分层：工作区/项目/系统配置 |
| **功能边界** | 模糊，全局与项目混杂 | 清晰，按层级划分 |
| **用户路径** | 困惑，不知道在哪里操作 | 明确，按场景入口 |
| **权限控制** | 粗粒度 | 细粒度（系统/项目/数据三级） |

### 6.2 预期收益

1. **降低用户学习成本**
   - 按使用场景组织功能
   - 明确的全局 vs 项目区分

2. **减少操作错误**
   - 避免在全局配置误以为项目配置
   - 清晰的权限提示

3. **提升工作效率**
   - "我的工作"聚合待办事项
   - 项目内一站式配置

4. **增强系统可维护性**
   - 清晰的模块边界
   - 明确的权限模型
