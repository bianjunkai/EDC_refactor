# EDC临床研究数据采集系统 - 完整UX设计与PRD文档

## 第一部分：用户研究

### 1.1 目标用户分析

EDC系统服务6类核心用户角色，每类用户有不同的目标、痛点和使用场景：

| 用户角色 | 核心目标 | 主要痛点 | 使用频率 |
|---------|---------|---------|---------|
| **研究机构管理员** | 系统初始化、用户权限分配、中心配置 | 多项目并行管理复杂、权限配置繁琐 | 每周 |
| **项目管理员/PI** | 项目立项、CRF设计、数据审核、PI签字 | 跨中心协调困难、数据质量把控 | 每天 |
| **数据录入员** | 快速准确录入CRF数据、处理质疑 | 表单复杂、重复录入、质疑闭环周期长 | 每天 |
| **审核人员** | 项目审核、质疑处理、数据确认 | 审核流程不透明、待办事项分散 | 每天 |
| **稽查人员** | 稽查留痕、数据导出、合规检查 | 追溯历史困难、导出配置复杂 | 每月 |
| **监查员(SDV)** | 源数据核查、SDV留痕 | 现场核查与系统录入脱节 | 每周 |

## 第三部分：核心业务流程

### 3.1 项目生命周期流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   立项中    │────▶│   审批中    │────▶│   进行中    │
│  (可编辑)   │     │ (等待审核)  │     │ (可录入)    │
└─────────────┘     └──────┬──────┘     └──────┬──────┘
      ▲                    │                    │
      │                    ▼                    ▼
      │              ┌─────────────┐     ┌─────────────┐
      └──────────────│   已驳回    │     │   已锁库    │
         (修改重提)  │  (可修改)   │     │ (只读状态)  │
                     └─────────────┘     └─────────────┘
```

**状态流转说明**：
1. **立项中**：创建项目后可编辑所有配置，保存但不提交
2. **审批中**：提交审核后锁定，等待各级审批
3. **已驳回**：审核不通过，返回修改，可重新提交
4. **进行中**：审核通过，可开始入组和数据录入
5. **已锁库**：PI签字后锁定，数据只读，可解锁（需权限）

### 3.3 质疑处理流程

```
质疑发起方                    质疑处理方
    │                            │
    │────── 发起质疑 ────────────▶│
    │    (填写质疑内容)           │
    │                            │
    │◀────── 收到通知 ────────────│
    │                            │
    │                            │────── 查看质疑 ────▶
    │                            │        (定位数据)
    │                            │◀──────
    │                            │
    │                            │────── 回复质疑 ────▶
    │                            │    (修改数据/解释)
    │◀────── 收到回复 ────────────│
    │                            │
    │────── 确认/追问 ───────────▶│
    │                            │
    │────── 关闭质疑 ────────────▶│
         (状态变为已解决)
```

**质疑状态流转**：
- 待处理 → 处理中（首次回复）
- 处理中 → 待确认（追问）
- 待处理 → 已解决（关闭）
- 待处理 → 已撤销（撤销）

### 3.6 AI Agent智能介入点

系统预留两个AI Agent能力接入点：

#### 接入点1：CRF设计智能助手（CRF AI Designer）

**触发时机**：
- 用户上传临床研究方案文档（Word/PDF）
- 用户在Excel模板中填写设计意图
- 用户在系统中输入文字描述

**AI能力**：
```
输入: 临床研究方案文档 / 文字描述
  │
  ▼
AI解析:
  • 识别研究阶段和访视时间点
  • 提取需要收集的数据点（人口统计学、病史、实验室检查等）
  • 识别数据类型和验证规则
  • 理解访视-检查项的关联关系
  │
  ▼
输出: 推荐的eCRF配置
  • 自动生成访视期结构
  • 推荐CRF表单和字段
  • 生成访视-CRF矩阵
  • 用户可编辑调整后确认
```

**UI交互**：
- "智能导入"按钮
- 上传文档 → 显示AI解析进度 → 展示预览 → 确认应用

#### 接入点2：OCR数据识别导入（OCR Data Import）

**触发时机**：
- 数据录入员在CRF录入页面点击"OCR导入"
- 批量上传纸质CRF扫描件

**AI能力**：
```
输入: CRF纸质表单扫描件/照片
  │
  ▼
OCR识别:
  • 表单版面分析（表格结构识别）
  • 手写/印刷文字识别
  • 勾选框状态识别
  • 数字字段识别
  │
  ▼
智能映射:
  • 匹配当前CRF表单字段
  • 字段级置信度评分
  • 低置信度字段高亮提示
  │
  ▼
输出: 结构化数据
  • 自动填充到对应字段
  • 人工核对界面
  • 确认后批量保存
```

**UI交互**：
- "OCR识别导入"按钮
- 上传图片 → 显示识别进度 → 分屏对照（原图|数据）→ 核对修正 → 确认导入

### 3.7 数据直接绑定功能（Data Binding）

**功能说明**：支持将外部数据源直接绑定到CRF表单字段，实现数据自动填充。

**使用场景**：
- 对接医院HIS/LIS系统，自动获取实验室检查结果
- 对接穿戴设备，自动获取生命体征数据
- 对接第三方数据平台

**配置流程**：
```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 配置数据视图（Data View）                             │
│  • 定义数据源连接（API地址、数据库连接字符串）                  │
│  • 配置数据查询（SQL/API参数）                                 │
│  • 测试连接并预览数据                                          │
│  • 保存数据视图配置                                            │
└──────────────────────────────┬──────────────────────────────┘
                               ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 配置字段映射（Field Mapping）                         │
│  • 选择目标CRF表单                                             │
│  • 选择数据源字段 → 映射到 → CRF字段                          │
│  • 配置转换规则（单位转换、编码映射、日期格式）                 │
│  • 配置触发条件（手动触发/定时同步/事件触发）                   │
└──────────────────────────────┬──────────────────────────────┘
                               ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 数据绑定执行                                          │
│  • 在CRF录入页面显示"数据绑定"按钮/自动同步标识                 │
│  • 获取外部数据并填充到对应字段                                │
│  • 标记自动填充字段（不同背景色/图标标识）                     │
│  • 人工确认或修改                                              │
│  • 提交时记录数据来源（手动录入/自动绑定）                     │
└─────────────────────────────────────────────────────────────┘
```

**数据视图管理**：
- 数据视图列表（名称、数据源类型、状态）
- 新增/编辑数据视图
- 测试连接功能
- 字段映射配置界面

## 第五部分：交互规范

### 5.1 全局交互模式

#### 页面加载
- **骨架屏**：列表页、详情页使用骨架屏占位
- **加载进度**：长耗时操作（导出、批量处理）显示进度条
- **懒加载**：图片、长列表滚动加载

#### 导航
- **面包屑**：层级>2时显示，可点击跳转
- **返回按钮**：详情页、创建页左侧显示返回按钮
- **标签页切换**：平滑过渡，保留编辑状态

#### 快捷操作
- **右键菜单**：表格行右键显示快捷操作
- **键盘快捷键**：
  - `Ctrl+S`：保存
  - `Ctrl+F`：搜索
  - `Esc`：关闭弹窗/抽屉

### 5.2 表单交互规范

#### 输入反馈
- **实时验证**：输入完成后立即验证（失焦时）
- **错误提示**：字段下方显示红色错误信息
- **成功提示**：验证通过显示绿色对勾（可选）

#### 自动保存
- **草稿保存**：CRF录入每30秒自动保存草稿
- **保存提示**：保存成功后显示"已自动保存"提示
- **恢复草稿**：重新进入时询问是否恢复草稿

#### 表单提交
- **提交验证**：提交前全表验证，错误项滚动到可视区域
- **提交确认**：重要操作二次确认
- **提交后**：成功后跳转，失败后保留表单数据

### 5.3 表格交互规范

#### 排序与筛选
- **表头排序**：点击表头排序，支持多列排序
- **筛选器**：表头下拉筛选，支持搜索
- **筛选标签**：顶部显示已选筛选条件，可一键清除

#### 行操作
- **悬停效果**：鼠标悬停高亮整行
- **行内操作**：主要操作直接显示，次要操作收拢
- **批量操作**：勾选多行后显示批量操作栏

#### 分页
- **页码**：居中显示，支持跳转
- **每页条数**：可选 10/20/50/100
- **总记录数**：右下角显示

### 5.4 反馈机制

#### 消息提示 (Toast)
| 类型 | 用途 | 持续时间 | 位置 |
|-----|-----|---------|-----|
| 成功 | 操作成功 | 3秒 | 右上 |
| 错误 | 操作失败 | 5秒 | 右上 |
| 警告 | 需要注意 | 4秒 | 右上 |
| 信息 | 普通提示 | 3秒 | 右上 |

#### 确认对话框
- **删除确认**："确定删除'XXX'吗？此操作不可恢复。"
- **状态变更**："确定将项目状态改为'进行中'吗？"
- **离开确认**："您有未保存的更改，确定要离开吗？"

#### 空状态
- **无数据**："暂无数据" + 图标 + 操作引导
- **无搜索结果**："未找到匹配的项目，请调整搜索条件"
- **无权限**："您没有权限访问此页面"

### 6.5 标准字典库管理（新增）

系统内置行业标准医学字典，支持项目引用和版本控制。

#### 支持的字典类型

| 字典 | 用途 | 数据来源 |
|-----|-----|---------|
| **CDISC** | 临床数据交换标准 | CDISC官方发布 |
| | - SDTM（研究数据表格模型） | |
| | - ADaM（分析数据模型） | |
| | - CDASH（数据获取标准） | |
| **WHODrug** | 药物编码 | WHO Drug Dictionary |
| | - Drug Name（药物名称） | |
| | - ATC Code（解剖治疗分类） | |
| **MedDRA** | 医学术语编码 | MedDRA词典 |
| | - SOC（系统器官分类） | |
| | - PT（首选术语） | |
| | - LLT（低位术语） | |
| **自定义字典** | 项目特殊需求 | 用户导入 |

#### 字典管理功能

**1. 字典导入**
- 支持Excel/CSV格式导入
- 自动识别字典类型
- 版本号管理（如MedDRA 25.0, WHODrug 2024Q1）
- 增量更新支持

**2. 字典浏览**
- 树形结构展示（如MedDRA: SOC → HLGT → HLT → PT → LLT）
- 搜索功能（编码、名称模糊搜索）
- 多语言支持（中英文对照）

**3. 项目字典引用**
- CRF设计时选择字典字段
- 数据录入时提供字典值搜索/选择
- 编码一致性校验

**4. 版本控制**
- 记录项目使用的字典版本
- 字典升级时提示影响分析
- 历史版本追溯

### 7.2 技术实现建议

1. **状态管理**：使用React Context + useState，暂不需要Redux
2. **权限控制**：前端路由守卫 + 组件级权限指令
3. **数据持久化**：localStorage存储草稿，避免数据丢失
4. **性能优化**：虚拟滚动处理大数据表格，懒加载非关键资源
5. **响应式**：桌面优先，基础响应式适配
